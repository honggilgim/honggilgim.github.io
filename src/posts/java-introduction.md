## 개요

스프링을 다루는 개발자로써 자바에 대한 기초 상식이 부족하다는 것을 느껴서 자바의 정석을 사서 정독하며 공부하기로 했다. 그래서 자바의 개념에 대해 적는 글을 써볼 예정이다.

## 자바의 특징

### 1. JVM (자바 가상 머신)

자바는 JVM을 통해 구동된다. JVM은 아쉽게도 운영체제에 따라 다르게 구동되지만 그 중심이 되는 자바는 같은 코드로 다른 운영체제에서도 실행된다. 운영 체제에 자바라는 서비스를 올리는 것이 아닌 자바 프로그램이 JVM위로 올라가 운영체제는 자바 가상 머신을 통해서 소통한다.

### 2. 객체지향언어

자바는 객체 지향 언어로 상속, 캡슐화, 다형성이 적용된 언어이다.

### 3. 쉽다

자바는 C++과 스몰톡이라는 두 언어의 장점을 취합하여 만들어진 언어로 배우기 쉽다는 평가를 받고 있다. 또한 간결하고 이해하기 쉬운 코드로 이루어져 재사용성과 유지보수의 용이성 등 많은 장점을 가진다.

### 4. GC (Garbage Collection)

자바는 GC가 자동으로 메모리를 관리해주기 때문에 메모리 관리를 따로 하지 않아도 된다. 이는 장점도 있고 단점도 있는 방법이다. 이 메모리 관리는 보통 운영체제에서 진행해주는데, 자바는 언어 자체에서 메모리 관리를 지원해준다. 그래서 장점이라만 생각할 수도 있지만, 자바 GC의 구동 방식은 추측을 통해 이 메모리는 사용하지 않을 것이라고 느꼈을 때 메모리를 회수하는 방식이기에 단점 역시 존재한다. 사용하는 메모리를 회수해버린다면... 버그가 발생하기 딱 좋은 환경이다.

### 5. 멀티쓰레드 지원

이 방식은 나중에 가상 스레드로까지 뻗어나간다. 멀티 쓰레드는 운영체제에 따라 구현방식도 다르고 작동방식도 다르지만, 자바는 훌륭한 멀티 쓰레드를 지원해주고 라이브러리를 지원해준다. 그리고 쓰레드에 대한 스케줄링은 자바 인터프리터가 담당한다. 멀티쓰레드는 Java의 stream()에서도 ParallelStream이라는 방식으로 지원해준다.

### 6. 동적 로딩을 지원한다

자바로 작성된 프로그램은 여러 개의 클래스로 구성되어 있는데, 실행 시 모든 클래스를 로딩하지 않고 필요한 시점에만 클래스를 로딩하여 사용한다. 이건 일부 클래스를 변경해도 전체 애플리케이션을 다시 컴파일하지 않아도 된다는 장점이 있다고 한다. 스프링을 다루는 개발자로써 톰캣의 hot deploy 같은 방식이라고 생각이 든다.

## 기초 코드

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello world!");
    }
}
```

인텔리제이로 처음 자바 프로젝트를 만들면 만들어주는 코드다. 여기서 `public static void main(String[] args) {}` 코드는 main메서드의 선언부이며 프로그램을 실행할 때 java.exe에 의해 호출될 수 있도록 약속한 부분이니 항상 똑같이 적어주어야 한다.

모든 클래스가 main메서드를 가질 필요는 없지만, 하나의 자바 어플리케이션에는 필수로 작성해주어야 한다.

이 main메서드 없이 자바는 동작하지 않는다. 하나의 소스 안에 클래스는 보통 하나만 정의하지만, 둘 이상의 클래스를 정의하는 것도 가능하다. 하지만 소스파일의 이름은 public class의 이름과 항상 일치해야 한다. 하나의 소스파일에 public class가 둘 이상은 존재하면 안된다. 대소문자도 맞춰야 한다.
