## 개요

자바의 근본인 객체지향에 대해 공부해보고자 한다.

## 객체지향의 장점과 개념

객체지향은 실제 사물과 비슷한 객체를 컴퓨터 세계에 만들어내기 위해 탄생한 개념이다. 실제 사물의 속성과 기능을 분석한 다음, 데이터와 함수로 정의함으로써 실제 세계를 컴퓨터 속에 옮겨 놓은 것과 같은 가상 세계를 구현한다.

> 실제 세계는 객체들로 이루어져 있으며, 발생하는 모든 현상은 이 객체들의 상호작용에 의한 것이다.

라는 개념에 중점을 두고 있으며, 상속, 캡슐화, 추상화 개념을 중심으로 점점 구체적으로 발전했다.

## 객체지향언어

객체지향언어는 기존의 프로그래밍 언어에서 몇 가지 규칙을 추가해 발전한 형태이다.

### 특징

1. 코드의 재사용성이 높다.
2. 코드의 관리가 용이하다.
3. 코드의 신뢰성을 높인다.

객체지향언어는 개발자에게 **코드의 재사용성이 높고 유지보수가 용이하다**라는 장점을 가져다 준다. 이러한 장점은 프로그램의 개발과 유지보수에 대한 비용을 획기적으로 감소시켜주었다.

객체지향에서 가장 중요한 점은, **재사용성과 유지보수 그리고 중복된 코드의 제거**라는 세 가지 관점이다. 이 세 가지를 익힌다면 스프링같은 프레임워크 및 상속, 다형성과 같은 기능을 이해할 때 더 쉽게 이해할 수 있다.

## 클래스와 객체

### 클래스와 객체의 정의와 용도

**클래스의 정의**: 클래스란 쉽게 객체를 정의해 놓은 것이다.

**클래스의 용도**: 클래스는 객체를 생성하는데 사용된다.

클래스는 쉽게 객체를 만드는 설계도이자 틀이고 객체는 클래스에서 적용된 대로 사용된다.

객체는,

> 실제로 존재하는 것. 사물 또는 개념

이라는 개념을 가지고 있으며 이에 맞추어 유형 객체인 노트북과 같은 것에서부터 무형 객체인 수학공식과 같은 것들이 존재한다.

## 객체와 인스턴스

클래스를 객체로 만드는 과정을 클래스의 **인스턴스화**라고 하며, 어떤 클래스로부터 만들어진 객체를 그 클래스의 **인스턴스**라고 한다. 인스턴스는 객체의 하위 개념으로 같은 의미이지만, 객체는 모든 인스턴스를 대표하는 포괄적인 의미를 가지고 인스턴스는 어떤 클래스로부터 만들어진 것인지를 강조하는 구체적인 의미를 가지고 있다.

## 객체의 구성 요소

객체는 속성과 기능이라는 두 종류의 구성요소로 이루어져 있다.

- **속성**: 멤버 변수, 특성, 필드, 상태
- **기능**: 메서드, 함수, 행위

```java
package com.example.wish.wishTicket.VO;

import lombok.Data;

@Data
public class RequestVO {
    // 순번
    public int seq;
    // 현재 상태
    public String currentState;
    // 진행 상태
    public String processState;
    // 코멘트
    public String comment;
    // 티켓 상태
    public String ticketStatus;
}
```

손쉽게 이해하기 위해 실제 객체 코드를 가져왔다. 이곳의 속성은 seq, currentState, processState, comment, ticketStatus가 있고 기능은 롬복의 @Data 어노테이션을 사용해서 가장 기초적인 메소드인 getter와 setter만 생성해 주었다.

## 인스턴스의 생성과 사용

이렇게 만든다고 하더라도, 우리는 실제로 객체를 생성해 인스턴스화해야 사용이 가능하다.

위의 예제를 사용해 실제 생성 코드를 짜보면,

```java
// 생성자를 통한 객체 생성
RequestVO requestVO = new RequestVO();
```

위 형식으로 생성해 사용이 가능하다.

여기서 `new RequestVO()`는, 객체라는 참조형 변수는 프로그램의 메모리 값만을 가지므로 실체를 넣어주어야 사용이 가능하다. 이 new 연산자를 통해 클래스의 인스턴스가 메모리의 빈 공간에 생성되고 각 변수들은 기본값으로 초기화된다. 그리고 우리는 미리 만들어둔 기능인 setter 메소드를 통해 값을 넣어줄 수가 있다.

## 프로그래밍 관점에서의 객체 (클래스)

클래스란 객체를 생성하기 위한 틀이며 클래스는 속성과 기능으로 정의되어 있다. 프로그래밍 관점에서 클래스는 데이터와 함수의 결합으로 이루어져 있다.

가장 대표적인 참조형 객체인 String 객체는 문자열을 다루기 위해 기존에 사용되던 char 배열 대신에 char배열 사용 당시 사용했던 여러 문자열들을 묶어놓은 클래스이다.

프로그래밍 관점에서 클래스는 크게

1. **데이터와 함수의 결합**
2. **사용자 정의 타입**

으로 볼 수 있다. 데이터와 함수의 결합은 위의 String 클래스에서 볼 수 있고,

**사용자 정의 타입**은 프로그래밍언어에서 제공하는 자료형 외에 프로그래머가 서로 관련된 변수들을 묶어 하나의 타입으로 만드는 사용자 정의 타입이다. 프로그램이란 곧 데이터들을 다루어 작업을 하는 일인데 이 프로그램에 사용되는 데이터들은 대다수 한 프로그램을 위한 데이터들로 쓰인다. 이 때 사용되는 데이터들을 묶어놓은 것이 사용자 정의 타입이다.
