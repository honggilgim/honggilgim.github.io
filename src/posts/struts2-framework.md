## 1. 개요

회사에서 웹 프로젝트로 스트럿츠 2 프레임워크를 활용한 개발을 하게 되었다. 주된 업무는 프론트단 홈페이지의 수정이지만, 스프링을 주로 사용하던 나에게 예전 버전인 스트럿츠 2 프레임워크는 경험해본 적 없는 프레임워크기에 공부해서 기록을 남겨본다.

## 2. 스트럿츠 2 프레임워크란?

Apache Struts 2는 자바를 활용한 오픈 소스 웹 애플리케이션 프레임워크이다.

홈페이지는 [https://struts.apache.org/](https://struts.apache.org/)에서 조회 가능하다.

스트럿츠 2 프레임워크는 MVC 모델(Model, View, Controller) 아키텍처를 기반으로 하는 프레임워크다. 오픈 소스 플랫폼으로 위의 홈페이지에서 조회가 가능하며, 웹 애플리케이션을 개발하는 일에 주로 사용된다. 사용자의 요청을 처리하는 요청 기반 프레임워크를 기반으로 개발했으며 요청 핸들러와 응답 핸들러가 존재한다. Ajax, REST 및 SOAP가 지원된다는 특징 또한 존재한다.

또한 주로 자바 기반의 JSP를 위해 많이 사용된다.

## 3. 특징

### POJO 기반의 동작

POJO(Plain Old Java Object, 쉽게 말해 순수 자바) 기반의 동작을 지원하며, POJO 기반 액션과 폼을 지원한다.

### Ajax 태그 지원

Ajax 태그를 지원한다.

### DI 지원

의존성 주입(Dependency Injection)을 제공한다.

### 다양한 표현식 언어 제공

JSTL과 같은 표현식 언어를 사용할 수 있다.

## 4. 스프링과의 차이

내가 메인 스택으로 사용하는 프레임워크이자 현재 넓리 사용되는 스프링과의 차이점이다.

둘 모두 MVC 패턴을 기반으로 이루어진 프레임워크기에 공통점이 많으나, 차이점 또한 많다.

### 경량 프레임워크 vs 무거운 프레임워크

스프링 자체는 가볍지 않다. 스프링은 대량의 코드로 이루어진 무거운 프레임워크다. 하지만, 스프링 프레임워크를 사용할 경우 코드는 가벼워진다. 이 코드의 최적화를 보고 스프링을 가볍다고 표현하고, 스트럿츠는 이 코드의 스타일로 인해 무겁다고 표현된다.

### 태그 라이브러리 지원 여부

스트럿츠는 태그 라이브러리를 지원하지만, 스프링은 지원해주지 않는다.

### 수동 코딩 지원 여부

스프링은 코딩 자체를 최소화한 경량화 프로젝트이지만 스트럿츠는 코딩을 수동으로 전부 입력해주어야 한다.

### 계층화된 아키텍처 여부

스프링은 계층화된 아키텍처를 지원하지만, 스트럿츠는 지원하지 않는다.
